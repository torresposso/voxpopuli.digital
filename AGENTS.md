# AGENTS.md

Compact, repo-specific guidance for coding sessions. Generic Laravel/WordPress/Sage knowledge is omitted.

## What this is

**voxpopuli.digital** — WordPress 6.9.4 on Bedrock, with a custom Sage 11 theme. Runs in Docker (FrankenPHP 8.4 + Node 20), uses SQLite. Spanish-language journalism CMS (Caribbean Colombia, progressive, research-driven). Editorial product context lives in `PRODUCT.md`, `PRD.md`, `BRANDBOOK.md`, `DESIGN.md` — read those before suggesting product/UX changes.

### Theme internals (`web/app/themes/voxpopuli/`)

- `functions.php` — boots Acorn, registers `App\Providers\ThemeServiceProvider` and `App\Providers\SeoServiceProvider`
- `app/Providers/ThemeServiceProvider.php` — extends `Roots\Acorn\Sage\SageServiceProvider`; binds `JsonLd` as singleton, overrides `SageVite` with custom `Vite`, registers 6 view composers (App, Post, Comments, FrontPage, Seo, Category)
- `app/Providers/SeoServiceProvider.php` — registers SEO-related bindings (`SeoMeta`, `MetaRenderer`, `JsonLd`, `Sitemap`)
- `app/setup.php` — `after_setup_theme` (menus, html5, post-thumbnails, sidebars) + block-editor asset injection + `theme_file_path` filter redirects `theme.json` → `public_path('build/assets/theme.json')` (auto-generated by Vite from Tailwind config)
- `app/filters.php` — WP filters
- `app/View/Composers/` — `App.php`, `Category.php`, `Comments.php`, `FrontPage.php`, `Post.php`, `Seo.php`
- `resources/views/{layouts,components,sections,partials}/` — Blade
- Template views: `404`, `archive`, `category`, `front-page`, `index`, `page`, `search`, `single`
- Components: `alert`, `badge`, `card`, `drawer`, `prose`
- Sections: `footer`, `header`, `sidebar`
- Partials: `comments`, `content`, `content-page`, `content-search`, `content-single`, `entry-meta`, `page-header`
- `vite.config.js` — Tailwind v4 plugin, Laravel Vite, `@roots/vite-plugin`; `base: /app/themes/sage/public/build/`
- `public/build/` — Vite output (gitignored)
- Theme has **no `config/` directory** (Sage 11 change from Sage 10)
- Root `<html>` uses `data-theme="voxpopuli"` (daisyUI custom theme defined in `resources/css/app.css`)

## Dev commands

Run **in this order** for a clean local boot:

```bash
docker compose up -d --build              # build images
docker compose up -d --force-recreate node   # recreate node if theme metadata changed
docker exec -w /app/web/app/themes/voxpopuli voxpopuli-app-1 composer install
docker exec -w /app/web/app/themes/voxpopuli voxpopuli-app-1 composer dump-autoload
```

Recurring:

| Action | Command |
|---|---|
| Lint (host) | `composer lint` (read-only) / `composer lint:fix` |
| Tests (host) | `composer test` — runs Pest |
| Theme dev (HMR) | `npm run dev` from root (proxies into theme) — or already running in `node` container on `:5174` |
| Theme build | `npm run build` from root |
| WP-CLI | `docker exec -w /app voxpopuli-app-1 wp <command>` (path set in `wp-cli.yml`) |
| Theme deps (inside container) | `docker exec -w /app/web/app/themes/voxpopuli voxpopuli-app-1 composer install` |

The `node` compose service runs `npm install && npm run dev` on every start. If you change `package.json` or theme `composer.json`, **recreate the container**, don't just restart.

## Quirks & gotchas

- **Sage 11 is a local clone** of `roots/sage`, not a composer package. `composer.json` only requires `roots/acorn`. Editing `vendor/roots/sage` does nothing — edit `web/app/themes/voxpopuli/` instead.
- **Theme has no `config/`** directory in Sage 11; view composers are wired in `ThemeServiceProvider::boot()`.
- **`App\` autoloader collision:** root `composer.json` maps `App\` → `web/app/themes/voxpopuli/app/`. Theme's own `composer.json` also maps `App\` → `app/`. They resolve to the same directory. Tests run from the root autoloader.
- **SQLite credentials in `.env` are dummies** (`DB_NAME='sqlite'`, etc.) — Bedrock's validator requires them, but `sqlite-database-integration` ignores them and uses `DB_DIR` instead.
- **`VITE_DEV_HOST`** (e.g. `192.168.1.33`) is the LAN IP for HMR from mobile devices. Update it in `.env` if your IP changes; Vite reads it at theme start.
- **Caddyfile** blocks `*.sqlite`, `*.db`, `/storage/database/*`, `/uploads/*.php`, `/.env*`, `/composer.*`, `/.git/*` at the edge. Don't move the DB into `web/`.
- **Composer runs in the `app` container** — host machine has no global `composer` or `php`. Use `docker exec -w /app voxpopuli-app-1 composer ...`.
- **Vite dev server runs in the `node` container** on `:5174` and the `app` container's Blade `@vite()` calls proxy through it. `public/build/` is built locally by the `node` container.
- **Theme is `11.x-dev`**, not a tagged release. Breaking changes between dev commits are possible. Pin in `style.css` if needed.
- **`bun.lock` exists in the theme** alongside `package-lock.json` — `package.json` uses npm scripts; bun.lock is incidental. Don't run `bun install`.
- **Block editor + `theme.json`** is auto-generated from Tailwind config by `@roots/vite-plugin`'s `wordpressThemeJson()`. Don't hand-edit `public/build/assets/theme.json`.
- **Root `<html>` has `data-theme="voxpopuli"`** — daisyUI custom theme defined in `resources/css/app.css` via `@plugin "daisyui/theme"`. The theme name in CSS must match.
