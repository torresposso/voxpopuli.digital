💡 What: Removed `update_post_meta` calls and the `vp_word_count` save operations from inside Blade components (`post-card`, `featured-card`, `content-search`) and `App\View\Composers\Post`.

🎯 Why: When rendering a list of posts missing the `vp_reading_time` metadata, the loop would fallback to computing the reading time on the fly and immediately writing it to the database using `update_post_meta`. This caused a severe N+1 write bottleneck on read-only rendering requests (a single view of an archive page could fire dozens of UPDATE queries). Since `web/app/themes/voxpopuli/app/setup.php` already hooks into `save_post` to calculate and properly persist this data when a post is created/edited, the frontend writes were completely redundant and actively harmful to TTFB.

📊 Measured Improvement:
- **Baseline (with update_post_meta inside loop):** ~0.021s per 1000 posts missing meta, plus the severe DB locking overhead of N+1 `UPDATE` statements on the frontend rendering thread (simulated in isolation, DB overhead is significantly higher on real infrastructure).
- **Optimized:** Reading the meta without inline writes prevents DB locking during loops entirely.
